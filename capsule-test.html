Comparison

CSS border-radius (Current)

✓ Pros

  • Simple and straightforward
  • Widely supported
  • Easy to understand
  • No complex SVG paths

✗ Cons

  • Limited to circular/elliptical corners
  • Can't create custom shapes
  • Issues with SVG overlays (your current problem)

Border-Image SVG Data URI (New)

✓ Pros

  • Scales perfectly at any size
  • No edge attachment issues
  • Can create any custom shape
  • Color embedded in SVG (semi-transparent support)
  • No additional DOM elements needed

✗ Cons

  • More complex CSS
  • Need to encode color in SVG path
  • Slightly less intuitive

Code Examples

Base Capsule Style (Keep the same SVG path!):

.capsule {
  border: 50px solid;
  border-image-source: url('data:image/svg+xml;charset=utf-8,<svg xmlns="http://www.w3.org/2000/svg" width="3" height="3" fill="%23FF6B6B"><path d="M2 0H1C1 .6.6 1 0 1v1c.6 0 1 .4 1 1h1c0-.6.4-1 1-1V1a1 1 0 0 1-1-1Z"/></svg>');
  border-image-slice: 1 fill;
}

To Make Cutouts Smaller (Recommended):

/* Just reduce the border-width! */
.capsule-small {
  border-width: 30px;  /* Smaller = smaller cutouts */
}

.capsule-tiny {
  border-width: 15px;  /* Even smaller */
}

Key insight: Keep the same SVG path, just adjust border-width to scale the cutout size. Don't modify the SVG path itself.

To change color: Update the fill attribute in the SVG (use URL-encoded hex like %23FF6B6B for #FF6B6B)
